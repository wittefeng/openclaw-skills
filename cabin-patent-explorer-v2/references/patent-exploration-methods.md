# 专利探索方法论

## 归档说明

本文档记录专利探索的改进方法论，基于用户反馈优化。

**更新日期：** 2026-04-23
**归档原因：** 用户反馈连续推荐方向未被采纳，分析原因并改进

---

## 问题分析

### 1. 查重机制失效
- 声称"未在存档中"，但实际已被车企做了
- 知识库滞后，无法获取最新信息
- 导致连续推荐已做方向

### 2. 创意度不够
- 方向多为"功能叠加"（A+B）
- 缺乏真正的范式转移
- 用户要的是"wow"不是"还可以"

### 3. 伪需求问题
- 很多方向解决的是"想象的需求"
- 不是用户真实的痛点
- 如"触觉互联网"——真的有这个需求吗？

### 4. 技术可行性虚高
- 声称"成本<¥500"，实际可能更高
- 声称"1年量产"，实际可能需要3-5年
- 缺乏工程验证

### 5. 同质化严重
- 10个方向本质相似（传感器+AI+执行器）
- 缺乏真正的差异化

## 三个改进方案

### 方案A：用户驱动模式

**适用场景：** 用户有明确痛点或场景

**执行流程：**
```
用户提供：
- 具体痛点（"长途驾驶腰背痛"）
- 或具体场景（"车内多人同时说话"）
- 或技术关键词（"压电材料"）

系统执行：
1. 定向查重（基于用户提供的关键词）
2. 确认新颖后生成方案
3. 明确标注"查重状态：通过/待验证"
```

**优势：**
- 基于真实需求
- 查重更精准
- 用户参与度高

**劣势：**
- 依赖用户输入
- 可能限制创意范围

---

### 方案B：标杆分析法

**适用场景：** 需要找出技术空白

**执行流程：**
```
研究对象：
- 特斯拉最新专利（USPTO检索）
- 蔚来/小鹏/理想发布会
- 奔驰/宝马/奥迪技术路线
- 博世/大陆/采埃孚供应商方案

分析维度：
1. 他们正在做什么？
2. 他们还没做什么？
3. 他们做不了的（技术壁垒）？

输出：
- 技术空白清单
- 差异化机会点
```

**优势：**
- 基于真实技术现状
- 避免重复造轮子
- 找到真正的空白

**劣势：**
- 需要持续跟踪
- 信息获取有延迟

**工具：**
- Google Patents
- 车企官网
- 行业报告

---

### 方案C：跨界移植法

**适用场景：** 寻找范式转移级创新

**执行流程：**
```
源领域研究：
- 医疗：健康监测、康复设备
- 航空：人机工程、安全系统
- 建筑：空间设计、材料创新
- 军事：极端环境、生存系统
- 体育：人体工学、性能优化

移植分析：
1. 源领域的成熟技术
2. 汽车场景的可行性
3. 车规化挑战
4. 成本/时间评估

输出：
- 跨界创新方向
- 技术移植路径
```

**优势：**
- 真正的范式转移
- 技术成熟度高
- 差异化明显

**劣势：**
- 车规化挑战大
- 成本可能较高

## 方案选择指南

| 场景 | 推荐方案 | 理由 |
|-----|---------|------|
| 用户有明确痛点 | A | 基于真实需求 |
| 需要找技术空白 | B | 基于标杆分析 |
| 需要范式转移 | C | 跨界创新 |
| 时间紧迫 | A | 最快出方案 |
| 追求高壁垒 | B+C | 技术空白+跨界 |
| 成本敏感 | A | 精准控制 |

## 执行检查清单

### 查重检查（必须）
```
□ 检索Google Patents（关键词组合）
□ 检索CNABS（中文专利）
□ 查阅车企最新发布会
□ 查阅供应商技术路线
□ 确认技术差异点
□ 明确标注"查重状态"
```

### 需求验证（必须）
```
□ 这是真实痛点吗？
□ 用户愿意为此付费吗？
□ 频次够高吗？
□ 现有方案为什么不够好？
```

### 可行性验证（必须）
```
□ 技术成熟吗？
□ 车规化可行吗？
□ 成本真实吗？
□ 时间真实吗？
□ 有工程验证吗？
```

### 差异化验证（必须）
```
□ 与现有方案有什么不同？
□ 是"更好"还是"不同"？
□ 技术壁垒在哪里？
□ 竞品多久能复制？
```
