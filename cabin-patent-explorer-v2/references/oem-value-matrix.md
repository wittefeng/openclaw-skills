# OEM价值评估矩阵

## 评估维度与权重

| 维度 | 权重 | 评分标准 | 数据来源 |
|------|------|---------|---------|
| 法规合规 | 40% | 强制要求程度 | 欧盟/国内法规 |
| 降本价值 | 30% | BOM降低幅度 | 成本测算 |
| 体验提升 | 20% | 用户满意度提升 | 用户研究 |
| 技术壁垒 | 10% | 复制难度 | 技术分析 |

## 详细评分标准

### 1. 法规合规（40%）

| 等级 | 分数 | 说明 | 示例 |
|------|------|------|------|
| 强制要求 | 10分 | 法规强制安装，不装不让卖 | DMS（欧盟）、儿童遗留检测（美国） |
| 法规跟进 | 8分 | 法规即将出台，提前布局 | 驾驶员状态监控（国内） |
| 标准推荐 | 6分 | 行业标准推荐，非强制 | 手势交互、语音控制 |
| 无法规 | 4分 | 纯体验优化 | 氛围灯联动、香氛系统 |

**法规清单（2026年）**：
- **欧盟**：DMS强制（2024）、儿童遗留检测强制（2025）
- **美国**：Hot Cars Act（儿童遗留）部分州强制
- **中国**：GB/T驾驶员状态监测（推荐，将转强制）
- **C-NCAP**：DMS加分项（2024版）

### 2. 降本价值（30%）

| BOM降低 | 分数 | 说明 | 示例 |
|---------|------|------|------|
| >40% | 10分 | 大幅降本，OEM强烈关注 | DMS+OMS一体化（省1颗摄像头+1颗SOC） |
| 30-40% | 8分 | 显著降本 | 传感器共享方案 |
| 20-30% | 6分 | 中等降本 | 算力复用方案 |
| 10-20% | 4分 | 小幅降本 | 线束优化 |
| <10% | 2分 | 降本有限 | 软件优化 |

**成本测算模板**：
```
原始成本：
- 硬件A：¥XXX
- 硬件B：¥XXX
- 软件授权：¥XXX
- 总计：¥XXX

优化后成本：
- 硬件A'：¥XXX（复用）
- 硬件B'：¥XXX（替代）
- 节省：¥XXX（-XX%）
```

### 3. 体验提升（20%）

| 提升幅度 | 分数 | 说明 | 量化指标 |
|---------|------|------|---------|
| >50% | 10分 | 革命性提升 | 唤醒率40%→85%（+112%） |
| 30-50% | 8分 | 显著提升 | 误报率15%→3%（-80%） |
| 20-30% | 6分 | 中等提升 | 响应时间500ms→300ms（-40%） |
| 10-20% | 4分 | 小幅提升 | 操作步数5→4（-20%） |
| <10% | 2分 | 体验微增 | UI优化 |

**体验指标库**：
- **安全类**：唤醒率、召回率、误报率
- **效率类**：任务完成时间、操作步数、响应延迟
- **满意度**：NPS评分、用户满意度（5分制）
- **使用率**：功能使用率、日活率

### 4. 技术壁垒（10%）

| 壁垒等级 | 分数 | 说明 | 技术特征 |
|---------|------|------|---------|
| 极高 | 10分 | 跨域协同+功能安全认证 | 座舱-智驾感知共享+ASIL D |
| 高 | 8分 | 多硬件协同+时序配合 | 振动+空调+转向联动 |
| 中高 | 6分 | 算法+硬件绑定 | 特定传感器+专用算法 |
| 中 | 4分 | 算法优化 | 纯算法改进 |
| 低 | 2分 | 易复制 | 通用算法+标准硬件 |

**壁垒论证模板**：
```
壁垒来源：
1. 硬件协同复杂度：[具体说明]
2. 算法训练数据：[数据壁垒]
3. 功能安全认证：[认证门槛]
4. 专利布局：[专利壁垒]

复制难度：
- 时间成本：X个月
- 资金成本：¥XXX万
- 技术门槛：[具体说明]
```

## 综合评分计算

```
综合评分 = 法规合规×40% + 降本价值×30% + 体验提升×20% + 技术壁垒×10%

评分等级：
- ≥8.5分：强烈推荐申请，OEM买单意愿强
- 7-8.5分：建议申请，需补强壁垒论证
- <7分：谨慎申请，可能壁垒不足或价值有限
```

## 典型方案评分示例

### 示例1：DMS+OMS一体化

| 维度 | 评分 | 说明 |
|------|------|------|
| 法规合规 | 10分 | DMS强制要求 |
| 降本价值 | 10分 | BOM降低40%（省1颗摄像头+1颗SOC） |
| 体验提升 | 6分 | 无感监控，体验微增 |
| 技术壁垒 | 8分 | 单目广角+共享特征+分支网络 |
| **综合** | **9.0分** | **强烈推荐** |

**OEM买单理由**：
- 法规强制，必须装
- 省1颗摄像头（¥150）+ 1颗SOC（¥80）= ¥230/车
- 年销10万辆 = 省¥2300万

### 示例2：疲劳多级干预

| 维度 | 评分 | 说明 |
|------|------|------|
| 法规合规 | 10分 | DMS强制要求 |
| 降本价值 | 4分 | 增加执行器成本 |
| 体验提升 | 10分 | 唤醒率40%→85%（+112%） |
| 技术壁垒 | 8分 | 多级置信度+时序配合 |
| **综合** | **8.2分** | **建议申请** |

**OEM买单理由**：
- 法规强制，且要求"有效干预"
- 单一声音告警无法满足法规"有效"要求
- 多级干预是法规合规的最优解

### 示例3：手势识别优化

| 维度 | 评分 | 说明 |
|------|------|------|
| 法规合规 | 4分 | 无强制要求 |
| 降本价值 | 2分 | 降本有限 |
| 体验提升 | 6分 | 识别率85%→92% |
| 技术壁垒 | 4分 | 纯算法优化 |
| **综合** | **3.8分** | **不建议申请** |

**问题**：
- 无法规驱动
- OEM不愿为体验优化买单
- 算法易被替代

## OEM决策流程模拟

```
OEM评估专利价值：

Q1: 是否法规强制？
    ├─ 是 → 必须买，进入Q2
    └─ 否 → 进入Q2

Q2: 能否降本？
    ├─ 能，且>20% → 愿意买，进入Q3
    └─ 不能/降本<20% → 进入Q3

Q3: 体验提升是否显著？
    ├─ 是，且>50% → 可能买，进入Q4
    └─ 否 → 进入Q4

Q4: 技术壁垒是否高？
    ├─ 是，别人绕不过去 → 愿意付高价
    └─ 否，易被复制 → 只愿意付低价

最终决策：
- 法规+降本+壁垒 → 高价买单（¥50-100万/专利）
- 法规+壁垒 → 中价买单（¥20-50万/专利）
- 降本+壁垒 → 低价买单（¥10-20万/专利）
- 体验+壁垒 → 谨慎买单（¥5-10万/专利）
- 无壁垒 → 不买或极低价格
```

## 注意事项

1. **法规是最大驱动力**：法规强制的专利价值最高
2. **降本要算清楚**：OEM对成本敏感，必须量化
3. **体验要可感知**：用户愿意为之付费的体验才有价值
4. **壁垒是议价筹码**：没有壁垒只能卖低价
5. **数据要真实**：OEM会验证，虚假数据损害信誉
