# 智能座舱专利方向库

## 已验证的专利方向（按框架格式）

### 方向一：感知仲裁
**由于** DMS摄像头说"疲劳"但方向盘握力说"正常"，**发生了** 多信号矛盾导致误报率高的问题，**系统通过** 感知置信度加权 + 场景权重动态分配，**调用了** 车速/ACC/时间的上下文数据，最终解决了 疲劳检测单一传感器的误触发问题。

### 方向二：功能安全OTA
**由于** 座舱OTA升级过程中部分功能失效，**发生了** DMS/eCall等安全功能掉线的合规风险，**系统通过** A/B双分区 + 升级前检查 + 升级中功能锁定，**调用了** 独立安全MCU和看门狗，最终解决了 OTA升级过程的功能安全问题。

### 方向三：音频隐私隔离
**由于** 副驾打电话时车内麦克风同时采集到音乐声，**发生了** 通话对方听不清的隐私投诉，**系统通过** 麦克风阵列 + 波束成形 + 声纹定位，**调用了** 实时声源分离算法，最终解决了 多声源场景下的语音清晰度问题。

### 方向四：DMS+OMS一体化
**由于** 座舱同时需要DMS（驾驶员监控）和OMS（乘员监控）两个独立系统，**发生了** 摄像头重复布置、算力浪费、成本居高不下的问题，**系统通过** 单目广角摄像头 + 共享特征提取 + 分支输出网络，**调用了** 同一颗ISP和SOC资源，最终解决了 双系统分离导致硬件冗余的问题。

### 方向五：故障自隔离架构
**由于** 座舱娱乐域进程崩溃时需要整车重启，**发生了** 音乐卡顿导致导航/仪表同步黑屏的安全隐患，**系统通过** 微内核虚拟化 + 域进程独立监控，**调用了** Hypervisor层和看门狗定时器，最终解决了 局部故障引发全局失效的问题。

### 方向六：毫米波手势+活体复用
**由于** 现有手势识别依赖ToF摄像头，在强光/脏污环境下失效，**发生了** 用户在夏季强光下无法用手势控制空调的问题，**系统通过** 60GHz毫米波雷达 + 手势轨迹补偿算法，**调用了** 雷达回波的微多普勒特征，最终解决了 视觉类手势识别的环境限制问题。

### 方向七：座舱-智驾感知共享
**由于** 座舱OMS摄像头和智驾侧向摄像头功能重叠，**发生了** 同一部位重复布置摄像头、整车BOM成本虚高的问题，**系统通过** 摄像头数据路由 + 目标级输出共享，**调用了** 时间同步协议和SOC间高速总线，最终解决了 感知资源重复浪费的问题。

### 方向八：疲劳干预策略
**由于** DMS检测到驾驶员疲劳，**发生了** 现有方案只有声音告警，驾驶员容易忽视或疲劳导致反应迟钝的问题，**系统通过** 疲劳置信度分级 + 干预策略联动，**调用了** 座椅通风/按摩/方向盘振动/空调出风口的执行资源，最终解决了 疲劳告警"叫不醒人"的形同虚设问题。

### 方向九：泊车DMS动态降级
**由于** 车辆处于低速泊车状态，**发生了** DMS系统依然按高速场景运行、算力浪费且误报率高的问题，**系统通过** 车速感知 + 泊车意图识别 + DMS模式动态切换，**调用了** 超声波雷达和座舱环视摄像头资源，最终解决了 DMS在泊车场景下的功能冗余问题。

### 方向十：物品遗忘追踪
**由于** 驾驶员下车后钥匙/手机落在车内，**发生了** 现有方案无法感知车内物品状态、驾驶员离开后才发现遗漏的问题，**系统通过** UWB精准定位 + 移动轨迹分析，**调用了** 手机和钥匙的UWB锚点数据，最终解决了 遗忘提醒滞后的"事后发现"问题。
