# 权利要求书模板

## 独立权利要求模板

### 系统类权利要求

```
1. 一种[系统名称]，其特征在于，包括：
   [传感器/检测单元]，用于[检测/采集][条件A]，输出[信号/数据]；
   [处理/控制单元]，与所述[传感器]连接，用于[步骤C的关键处理]，
     当[条件A的量化阈值]时，输出[控制指令]；
   [执行单元]，与所述[处理单元]连接，用于根据所述[控制指令]
     [调用资源D的具体动作]，以解决[问题E]。

其中，所述[步骤C的关键处理]包括：
   [子步骤1：具体算法/逻辑]；
   [子步骤2：具体算法/逻辑]；
   [子步骤3：具体算法/逻辑]。

其中，所述[资源D]的参数为：[具体参数范围]。
```

**示例（疲劳多级干预）**：
```
1. 一种驾驶员疲劳多级干预系统，其特征在于，包括：
   DMS摄像头，用于采集驾驶员面部图像，输出疲劳置信度；
   控制器，与所述DMS摄像头连接，用于根据所述疲劳置信度
     进行多级判定，当置信度≥0.7时输出一级干预指令，
     ≥0.85时输出二级干预指令，≥0.95时输出三级干预指令；
   座椅振动执行器，与所述控制器连接，用于根据所述一级干预指令
     以8-15Hz频率振动座椅；
   空调风门电机，与所述控制器连接，用于根据所述二级干预指令
     在200ms内开启最大档制冷风；
   电动助力转向系统，与所述控制器连接，用于根据所述三级干预指令
     触发安全停车请求。
```

### 方法类权利要求

```
1. 一种[方法名称]，其特征在于，包括以下步骤：
   S1：[检测/获取][条件A的具体参数]；
   S2：判断[条件A的量化阈值]，若满足则进入S3；
   S3：[步骤C的关键处理]，包括：
       S3-1：[子步骤1]；
       S3-2：[子步骤2]；
   S4：[调用资源D]，以[具体参数]执行[动作]；
   S5：解决[问题E]，实现[效果F的量化指标]。
```

**示例（DMS+OMS一体化）**：
```
1. 一种座舱乘员监控方法，其特征在于，包括以下步骤：
   S1：通过单目广角摄像头采集座舱图像，视场角≥120°；
   S2：对所述图像进行共享特征提取，输出特征图；
   S3：将所述特征图分别输入驾驶员监控分支和乘员监控分支；
   S4：所述驾驶员监控分支输出驾驶员状态信息；
   S5：所述乘员监控分支输出乘员状态信息；
   其中，所述共享特征提取采用ResNet-18网络，输入分辨率为224×224。
```

## 从属权利要求模板

### 递进式细化

```
2. 根据权利要求1所述的[系统/方法]，其特征在于，
   所述[条件A]包括：[具体子条件1]和[具体子条件2]。

3. 根据权利要求1所述的[系统/方法]，其特征在于，
   所述[步骤C]还包括：[补充步骤]。

4. 根据权利要求1所述的[系统/方法]，其特征在于，
   所述[资源D]的参数为：[更窄的参数范围]。

5. 根据权利要求1所述的[系统/方法]，其特征在于，
   所述[效果F]为：[更具体的量化指标]。
```

**示例**：
```
2. 根据权利要求1所述的驾驶员疲劳多级干预系统，其特征在于，
   所述DMS摄像头在低功耗模式下以5fps采集图像，功耗<50mW。

3. 根据权利要求1所述的驾驶员疲劳多级干预系统，其特征在于，
   所述控制器还用于：在输出所述一级干预指令前，
   判断车速>60km/h且持续时间>30min。

4. 根据权利要求1所述的驾驶员疲劳多级干预系统，其特征在于，
   所述座椅振动执行器的频率为10-12Hz，响应时间<100ms。

5. 根据权利要求1所述的驾驶员疲劳多级干预系统，其特征在于，
   所述安全停车请求包括：逐级降低车速至60km/h以下，
   同时开启双闪警示灯。
```

## 权利要求布局策略

### 多层次保护

```
独立权利要求1（最大保护范围）
    ↓
从属权利要求2-3（核心特征细化）
    ↓
从属权利要求4-6（参数范围限定）
    ↓
从属权利要求7-10（替代实施例）
```

**布局原则**：
1. **独立权利要求**：采用"最小必要技术特征"，保护范围最大化
2. **从属权利要求2-3**：保护核心创新点，防止被规避
3. **从属权利要求4-6**：细化参数，为审查意见提供退守方案
4. **从属权利要求7-10**：保护替代实施例，扩大保护范围

### 多套权利要求布局

```
系统权利要求（产品保护）
方法权利要求（方法保护）
用途权利要求（应用保护）

示例：
1. 一种驾驶员疲劳多级干预系统...（系统）
10. 一种驾驶员疲劳多级干预方法...（方法）
20. 如权利要求1所述系统在乘用车中的应用...（用途）
```

## 常见错误与避免

### 错误1：权利要求过宽

```
❌ 错误：
"控制器用于智能地处理数据"

✅ 正确：
"控制器用于根据疲劳置信度进行多级判定，
 当置信度≥0.7时输出一级干预指令..."

避免：使用"智能地""高效地""优化地"等模糊词
```

### 错误2：缺少必要技术特征

```
❌ 错误：
只写"座椅振动"，不写频率范围

✅ 正确：
"座椅振动执行器以8-15Hz频率振动座椅"

避免：缺少具体参数，导致无法实施
```

### 错误3：纯算法权利要求

```
❌ 错误（客体不适格）：
"一种疲劳检测算法，包括：输入图像、提取特征、分类输出"

✅ 正确：
"一种驾驶员疲劳多级干预系统，包括：DMS摄像头、控制器、
 座椅振动执行器..."

避免：必须结合硬件，不能纯算法
```

### 错误4：引用关系混乱

```
❌ 错误：
"根据权利要求3所述的系统，其特征在于..."
（权利要求3是从属权利要求，不能作为引用基础）

✅ 正确：
"根据权利要求1或3所述的系统，其特征在于..."
（引用独立权利要求1，或并列引用3）

避免：只能引用独立权利要求，或引用在前的权利要求
```

## 权利要求书撰写检查清单

```
□ 独立权利要求包含"最小必要技术特征"
□ 避免了模糊词（"高效""智能""优化"）
□ 包含了具体参数和阈值
□ 结合了硬件和软件
□ 从属权利要求逐级细化
□ 引用关系正确
□ 多套权利要求布局完整
□ 保护范围合理，不过宽也不过窄
```

## 完整权利要求书示例

**方案**：DMS+OMS一体化系统

```
1. 一种座舱乘员监控系统，其特征在于，包括：
   单目广角摄像头，视场角≥120°，用于采集座舱图像；
   处理器，与所述摄像头连接，用于对所述图像进行共享特征提取，
     输出特征图；
   驾驶员监控分支网络，与所述处理器连接，用于根据所述特征图
     输出驾驶员状态信息；
   乘员监控分支网络，与所述处理器连接，用于根据所述特征图
     输出乘员状态信息；
   其中，所述共享特征提取采用ResNet-18网络，输入分辨率为224×224。

2. 根据权利要求1所述的系统，其特征在于，
   所述单目广角摄像头的分辨率为1920×1080，帧率为30fps。

3. 根据权利要求1所述的系统，其特征在于，
   所述驾驶员监控分支网络包括：
   注意力检测子网络，用于检测驾驶员注意力状态；
   疲劳检测子网络，用于检测驾驶员疲劳状态；
   分心检测子网络，用于检测驾驶员分心状态。

4. 根据权利要求1所述的系统，其特征在于，
   所述乘员监控分支网络包括：
   儿童检测子网络，用于检测儿童是否遗留；
   姿态检测子网络，用于检测乘员姿态；
   情绪检测子网络，用于检测乘员情绪状态。

5. 根据权利要求1所述的系统，其特征在于，
   所述处理器还用于：根据车速动态调整所述共享特征提取的复杂度，
   当车速<30km/h时降低特征提取层数至12层，当车速≥30km/h时
   恢复至18层。

6. 根据权利要求1所述的系统，其特征在于，
   所述驾驶员监控分支网络和所述乘员监控分支网络共享
   部分卷积层权重。

7. 一种座舱乘员监控方法，其特征在于，包括以下步骤：
   S1：通过单目广角摄像头采集座舱图像；
   S2：对所述图像进行共享特征提取，输出特征图；
   S3：将所述特征图分别输入驾驶员监控分支和乘员监控分支；
   S4：所述驾驶员监控分支输出驾驶员状态信息；
   S5：所述乘员监控分支输出乘员状态信息。

8. 根据权利要求7所述的方法，其特征在于，
   所述共享特征提取采用ResNet-18网络，输入分辨率为224×224。

9. 根据权利要求7所述的方法，其特征在于，
   还包括：根据车速动态调整所述共享特征提取的复杂度。

10. 如权利要求1-6任一项所述系统在乘用车中的应用。
```

**布局分析**：
- 权利要求1：独立权利要求，最大保护范围
- 权利要求2-6：从属权利要求，逐级细化
- 权利要求7-9：方法权利要求，保护方法
- 权利要求10：用途权利要求，保护应用
